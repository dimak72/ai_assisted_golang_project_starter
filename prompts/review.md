# Code Review

Review the current implementation for correctness, maintainability, and test coverage.

Prioritize findings that could cause incorrect behavior, regressions, confusing APIs, brittle tests, missing edge cases, or avoidable complexity.

For each finding, include:

- Severity
- File and line reference
- Why it matters
- Suggested fix

Keep the review focused on actionable issues. If no issues are found, state that clearly and mention any remaining test gaps or residual risk.
